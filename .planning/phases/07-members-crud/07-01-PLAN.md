## Plan 07-01: Members CRUD Infrastructure

<objective>

Implementar infraestrutura de persistência e componentes UI para operações CRUD de membros.

</objective>

<read_first>

- src/pages/MembersPage.jsx (current members page structure)
- src/components/auth/LoginForm.jsx (form pattern reference)
- src/components/layout/SidebarLayout.jsx (layout structure)
- .planning/REQUIREMENTS.md (v2.0 requirements)
- .planning/phases/07-members-crud/07-CONTEXT.md (design decisions)

</read_first>

<action>

1. **Criar storage service:**
   - Arquivo: `src/services/storage.js`
   - Funções: `getMembers()`, `saveMember(member)`, `updateMember(id, data)`, `deleteMember(id)`, `bulkDeleteMembers(ids)`
   - Usar chave `'siltec_members'` no localStorage

2. **Criar componente Modal:**
   - Arquivo: `src/components/ui/Modal.jsx`
   - Props: `isOpen`, `onClose`, `title`, `children`
   - Estilo: glassmorphism (backdrop blur, bg-white/10)
   - Overlay com click para fechar

3. **Criar componente Toast:**
   - Arquivo: `src/components/ui/Toast.jsx`
   - Tipos: success, error, info
   - Auto-dismiss após 3 segundos
   - Posição: top-right

4. **Criar MemberForm (Create):**
   - Arquivo: `src/components/crud/MemberFormCreate.jsx`
   - Campos: nome (required), email (required, unique), telefone, departamento (dropdown), status (ativo/inativo)
   - Validação: email válido, email único
   - Estilo glassmorphism igual LoginForm

5. **Criar MemberForm (Edit):**
   - Arquivo: `src/components/crud/MemberFormEdit.jsx`
   - Mesmo schema do Create, pré-preenchido com dados existentes
   - Atualiza via storage.updateMember()

6. **Criar ConfirmDialog:**
   - Arquivo: `src/components/ui/ConfirmDialog.jsx`
   - Props: isOpen, title, message, onConfirm, onCancel
   - Usado para delete e bulk-delete

7. **Atualizar MembersPage:**
   - Substituir mock data por storage.getMembers()
   - Adicionar botão "Cadastrar Membro" no header
   - Adicionar botão Editar em cada linha da tabela
   - Adicionar checkbox para seleção múltipla
   - Adicionar botão Excluir (com Confirmation)
   - Integrar modais e toast

8. **Migrar dados existentes:**
   - Ao carregar pela primeira vez, converter mock generateMockMembers() para storage

</action>
