## Phase-Specific UI Decisions (from CONTEXT.md)

### Login Page (`/login`)
- Layout: Centralizado, card glassmorphism com largura máxima 400px
- Campos: E-mail (type=email), Senha (type=password com toggle visibilidade)
- Checkbox: "Lembrar-me" (persistência via localStorage)
- Link: "Esqueceu sua senha?" → `/recover-password`
- Mensagens de erro abaixo do formulário (texto vermelho, ícone de alerta)

### Password Recovery Page (`/recover-password`)
- Layout: Similar ao login, card glassmorphism
- Campo: E-mail para recuperação
- Após submit: Exibir mensagem "E-mail enviado para ***@***.com" (simulação MVP)
- Link: "Voltar ao login" → `/login`

### Route Guards & Access Control
- Componente `RequireRole` wrapper: `<RequireRole role="admin"><Component /></RequireRole>`
- Redirecionamento: Usuários sem permissão → `/access-denied`
- Líderes no Financeiro: Botões de "Editar", "Excluir" desabilitados (disabled + tooltip "Apenas visualização")

### Token Handling (JWT)
- JWT armazenado em localStorage (se "Lembrar-me") ou sessionStorage (caso contrário)
- Payload inclui: `{ email, role, name }`
- Leitura do token via `jwt-decode` (simples decodificação client-side)
